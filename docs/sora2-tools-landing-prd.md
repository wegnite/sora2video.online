# Sora2 工具站落地页 PRD

## 1. 背景与目标
- **背景**：sora2video.online 已完成首页品牌升级，但 “精品工具” 页面仍沿用旧的通用工具文案与结构，缺乏对 sora2 工具生态、工作流与付费路径的清晰叙述。
- **核心目标**：重构工具页为完整的工具站落地页，展示关键产品能力、引导用户体验免费功能并完成付费转化，强化 sora2 作为专业视频自动化工具集的定位。
- **成功标准**：
  - 用户能够理解四大工具分区（生成、编排、协作、数据）以及对应价值。
  - 免费试用与付费方案路径清晰，支付组件可用。
  - 页面内容符合 sora2 品牌语调，SEO 关键词聚焦 “sora2 工具 / sora prompt / sora2 automation”。

## 2. 用户画像
| 角色 | 需求 | 关键痛点 |
| ---- | ---- | -------- |
| 创意总监/制片人 | 需要快速预览工具覆盖范围与协作能力 | 缺乏端到端工作流、审批工具割裂 |
| 市场运营/品牌经理 | 找到标准化活动模板、掌握上线节奏 | 多渠道上线成本高、缺少规范 | 
| 技术/合规伙伴 | 关注安全与治理、接入成本 | 需要明确的数据安全与权限策略 |

## 3. 整体价值主张
- **一句话**：Sora2 工具站帮助跨职能团队用 sora prompt 驱动视频生产，从创意到上线在同一套自动化工具内闭环。
- **价值支柱**：
  1. **工具集成**：生成、编辑、调度、数据监控一站式覆盖。
  2. **协作治理**：角色权限、审批流、合规日志完整可追溯。
  3. **商业化准备**：定价结构支持订阅与一次性买断，结合信用点数扩展用量。

## 4. 页面信息架构
1. **Hero**：sora2 工具站定位、核心 CTA（立即体验 / 请求演示）。
2. **工具分区概览**：四块卡片展示生成中心、故事编排、协作中心、数据透视，每块含关键能力与深入链接。
3. **工作流演示**：时间线形式介绍 “构思 → prompt → 审批 → 发布” 四阶段，配合视觉资产。
4. **工具详情段落**：分组介绍具体工具（例如 Prompt Lab、Scene Composer、Delivery Console、Insight Grid），包含亮点、适用场景。
5. **客户案例/行业方案**：3-4 条行业案例，突出 sora2 工具如何落地。
6. **定价与支付**：引用现有 PricingSection，强调订阅（月/年）、终身买断、信用点包的差异与支付能力。
7. **常见问题**：聚焦工具接入、团队权限、支付退款。
8. **二次 CTA**：强化预约演示与立刻上手。

## 5. 功能需求清单
- **内容模块**
  - Hero 模块：主标题、描述、副 CTA、背景渐层。
  - 工具分区组件：四列卡片，支持图标、要点列表、内链。
  - 工作流步骤：可复用 timeline/stepper 组件，展示阶段与产出。
  - 工具详情网格：大图文块 + 次要强调。
  - 行业方案段落：卡片含指标、关键成果。
  - 定价卡片：复用 `PricingSection`，确保 CTA 与付款流程连接 Stripe。
  - FAQ：使用手风琴组件展示 4-5 条问答。
- **国际化**：新增/更新 `messages/en.json` 与 `messages/zh.json` 中 `ToolsPage` 命名空间内容。
- **路由与导航**：保留 `/[locale]/tools` 访问路径，导航项若存在需同步更新标题。
- **SEO**：
  - 页面关键词：sora2 工具站、sora prompt 工具、sora2 automation platform。
  - Hero 描述与工具模块文案需自然重复 sora 相关词，密度约 4-5%。
- **支付**：
  - 页面中确保调用 `PricingSection`，底层依赖 Stripe Provider。
  - CTA 按钮（Hero、二次 CTA）链接到 `/pricing` 锚点或触发滚动。
  - 保持 `PricingSection` 中订阅/一次性购买按钮现有逻辑。

## 6. 数据 & 埋点诉求
- **关键指标**
  - Tools 页访问-到-定价表曝光率（滚动触达比例）。
  - CTA 点击率：立即体验（跳转至工具登录/注册）、预约演示（邮件或表单）。
  - Stripe 支付转化（保留现有事件上报）。
- **埋点建议**（后续实施）
  - `tools.hero.cta_primary_click`
  - `tools.hero.cta_secondary_click`
  - `tools.section.open`（记录用户展开 FAQ）

## 7. 交互与视觉要求
- 深色背景延续首页风格，强调蓝紫渐层与玻璃拟态卡片。
- 卡片 hover 态增加轻微放大与辉光。
- 响应式：
  - Mobile：模块顺序保持，上下堆叠，CTA 固定于段落末尾。
  - Desktop：工具分区与案例使用 2-4 列栅格。
- 使用现有 UI 组件库（Card、Button、Badge 等），避免引入新的依赖。

## 8. 技术实现概述
- 页面仍为服务端组件，使用 `getTranslations` 获取多语言文案。
- 新建/更新局部展示组件（如工具卡片、流程步骤）放置于 `src/components/marketing/tools` 目录。
- 保持 lint、类型检查通过。
- 复用 `LocaleLink` 实现跨语言的 CTA 跳转。
- 验证支付：沿用 `PricingSection`，确保按钮触发 `getPaymentProvider().createCheckoutSession` 相关流程。

## 9. 范围外事项
- 不改动 Stripe 后端逻辑与 webhook。
- 不在本次迭代内新增交互式演示或视频组件。
- 不实现新的埋点事件发送，仅预留结构与文案。

1. `docs/sora2-tools-landing-prd.md`（本文件）。
2. 重构后的 `/tools` 页面及相关组件。
3. 更新后的中英双语文案。
4. 保证 `PricingSection` 正常显示并可进入支付流程。
